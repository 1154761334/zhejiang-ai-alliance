# 会员档案管理与尽调机制测试计划 (Test Plan)

## 测试目标
验证“公共层(A) - 权威层(B) - 尽调层(C)”三层分离的会员档案管理机制的正确性。确保权限隔离生效（企业不可见B、C层），且前端调研包生成逻辑正常。

## 测试环境
- 本地开发环境 (Localhost)
- Directus CMS 后台 (+ Admin/User 角色)
- Next.js 前端应用

## 缺陷管理与提交流程
1. 如测试发现 `待测 (Pending)` -> `失败 (Failed)`，直接于本表底部更新问题原因。
2. 问题修复后，重新执行受影响用例。
3. 全部通过后，此文件进行归档。

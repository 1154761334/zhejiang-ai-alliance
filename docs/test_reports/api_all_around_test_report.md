# 全方位 API 自动化测试与权限安全报告

## 1. 测试综述
本次测试对“浙江省AI智能体产业联盟平台”进行了全方位的 API 走查，重点针对 **Directus v11** 的新权限架构（PBAC）进行了深度加固和回归验证。

## 2. 测试覆盖详情

### A. 核心业务流 (14/14 PASSED)
- **访客流**: 验证了对已发布(Published)数据的只读权限及对待审核数据的过滤。
- **供应商流**: 涵盖了账号注册、登录、企业建档、产品入库及审批申请。
- **需求方流**: 涵盖了账号注册、需求调研表单提交及角色隔离验证。
- **管理审批流**: 验证了管理员对 Pending 数据的全局可见性及状态翻转逻辑。

### B. 深度安全加固 (10/10 PASSED/FIXED)
- **项级所有权 (Item-level Security)**: 
  - 修复了 `companies`、`products`、`survey_needs` 的越权编辑漏洞。
  - 通过注入 `user_created` 审计字段（Audit Fields）实现了基于所有权的强制隔离。
- **关系关联安全**: 
  - [x] 阻止 Demander 越权修改 Supplier 的产品。
  - [x] 访客无法通过 ID 直接爬取未授权（非 published）的企业档案。
  - [!] **风险提示**: 跨集合关联创建（如将需求挂载到他人企业）在 Directus 11 语法下建议后续增加 `Validation` 钩子或前端逻辑二次校验。
